Problems Found & Solutions Applied
PROBLEM 01 — CRITICAL · Severe Class Imbalance
98.7% benign vs 1.3% attack in raw data (2.65M benign, 34K attack). A naive model achieves 98.7% accuracy by predicting all-benign — learning nothing useful. Standard training is completely useless.
SOLUTION — Two-Phase Rebalancing (Downsample + SMOTE)
Downsample benign 2.65M → 75,600 (25K per major protocol). Then SMOTE each attack class to 12,000 samples with k_neighbors=3. SMOTE generates synthetic neighbors rather than exact copies — avoids overfitting while filling rare classes.
PROBLEM 02 — Protocol-Specific NaN (up to 80%+)
Each log type only populates its own protocol fields. MQTT logs have NaN in all HTTP/CoAP columns and vice versa. Up to 80%+ NaN columns — expected by design, not data corruption.
SOLUTION — Context-Aware Fill with 0
Fill protocol-specific NaN with 0 — semantically correct. Zero HTTP status code means "no HTTP interaction." An MQTT log doesn't have an http_method because it's not HTTP, so 0 is the right representation.
PROBLEM 03 — CSV Parsing Errors
"Expected 48 fields, saw 52." HTTP user-agent strings containing commas (e.g. "Nmap Scripting Engine") were not properly quoted. Some Elasticsearch indices also returned extra unexpected fields beyond the expected schema.
SOLUTION — QUOTE_NONNUMERIC + Strict Column Filter
Added csv.QUOTE_NONNUMERIC to handle embedded commas in string fields. Added strict column filtering in export_data.py — only columns in the expected COLUMN_ORDER are kept, extra ES fields silently ignored.
PROBLEM 04 — Data Leakage (100% fake accuracy)
is_internal and is_known_device are perfect proxies — all benign is internal+known, all attacks are external+unknown. Model achieved 100% with zero real learning. SMOTE also injected "synthetic" string markers as categorical values, creating artifact one-hot columns.
SOLUTION — Drop Leaky Columns + Neutralize SMOTE Artifacts
Dropped is_internal and is_known_device entirely. Replaced all "synthetic" SMOTE markers with np.nan before one-hot encoding to prevent artifact columns like mqtt_action_synthetic. After fix: accuracy dropped from fake 100% → realistic 99%.
PROBLEM 05 — Extremely Rare Attack Classes
malware_download (1 sample) and post_exploit (20 samples) — too few for SMOTE which requires k_neighbors + 1 minimum. Cannot generate meaningful synthetic data from near-empty classes.
SOLUTION — Merge Into Parent Class
malware_download → exploit · post_exploit → exploit. After merging, all classes have ≥30 real samples — enough for SMOTE with k_neighbors=3. Semantically correct: both are exploitation-phase activities.
PROBLEM 06 — Extreme Outliers Distorting Scaling
payload_size, bytes_sent, bytes_received had extreme right-skew. Single entries with bytes_sent > 1 GB (logging errors or DDoS floods). MinMaxScaler compressed 99% of values into a near-zero band.
SOLUTION — 99th Percentile Clipping
Cap all numeric columns at 99th percentile using np.clip before feature engineering. Preserves the overall distribution shape while preventing extreme values from distorting the entire feature space.

Future Work — The Cat and Mouse Game
// The Perpetual Adversarial Loop
The current result — defensive AI neutralising the RL attacker on the first step every time — is not the end. It is the beginning of a cat and mouse game. Each time the defensive models improve enough to beat the offensive agent, the offensive agent can be retrained on the new defensive model, learning new evasion strategies. Then the defensive models retrain on the new adversarial traffic, becoming stronger again. This cycle continues indefinitely.
ROUND N — DEFENSIVE WINS
Current state: XGBoost + Autoencoder detects RL agent at step 1. Detection rate 100%. Defensive models are stronger.
⇄
ROUND N+1 — RETRAIN OFFENSIVE
RL agent retrains against the updated defensive models. Learns new evasion strategies, novel attack sequences, lower-noise primitives.
⇄
ROUND N+2 — RETRAIN DEFENSIVE
Defensive models retrain on the new adversarial traffic generated by the improved RL agent. Detection improves again. Loop continues.
